Reset Uptime Statistics
What this does:
  • Sets a new uptime calculation start point to now
  • All uptime percentages (24h, 7d, 30d, all-time) will count from this moment forward
  • The consecutive_down_count counter is reset to 0
What is preserved (audit integrity):
  • All PingHistory records remain — nothing is deleted
  • This reset action is logged with your username, timestamp, and reason
  • Previous uptime values are snapshotted for the audit trail
  • NetBox's change log also captures this change independently
When to use this: An IP address has been reassigned to a different device (e.g. old VM decommissioned, new device took over the IP). Resetting avoids polluting SLA reports with downtime from the previous owner.
This reset will be visible in the Reset Audit Log tab below, accessible via the REST API at /api/plugins/netbox-ping/uptime-resets/, and in NetBox's change log for this IP address.
